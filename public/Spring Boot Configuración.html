Refactorizando el código para <script setup>

El código se reescribe de la siguiente forma para aprovechar el <script setup>:

Código refactorizado con <script setup>

vue
<template>
  <div id="app">
    <div v-if="weatherData">
      <h1>Clima en {{ weatherData.name }}</h1>
      <p>Temperatura: {{ weatherData.main.temp }} °C</p>
      <p>Descripción: {{ weatherData.weather[0].description }}</p>
      <p>Humedad: {{ weatherData.main.humidity }} %</p>
      <p>Viento: {{ weatherData.wind.speed }} m/s</p>
    </div>

    <div v-else>
      <p>Cargando...</p>
    </div>
  </div>
</template>

<script setup>
import { ref, onMounted } from "vue";
import axios from "axios";

// Definición de datos reactivos
const weatherData = ref(null);
const city = ref("Madrid");
const apiKey = ""; // Agrega tu API key aquí

// Función para obtener el clima
const getWeather = async () => {
  try {
    const response = await axios.get(
      `https://api.openweathermap.org/data/2.5/weather?q=${city.value}&units=metric&appid=${apiKey}`
    );
    weatherData.value = response.data;
  } catch (error) {
    console.error("Error al obtener el clima:", error);
  }
};

// Llamada a la API cuando el componente se monta
onMounted(() => {
  getWeather();
});
</script>

<style scoped>
#app {
  font-family: Avenir, Helvetica, Arial, sans-serif;
  text-align: center;
  color: #2c3e50;
}

h1 {
  margin-top: 20px;
}

p {
  font-size: 18px;
}
</style>

Cambios clave:

  1. Importación de ref y onMounted:
    Usamos ref para definir variables reactivas, y onMounted para llamar a la API cuando el componente se haya montado (es decir, cuando la página esté lista).

  2. Definición de datos reactivos:

    • Usamos ref para weatherData, city y apiKey.
    • Los datos reactivos permiten que los cambios en el estado se reflejen automáticamente en el DOM.

  3. Eliminación de data, created, y methods:

    • No es necesario usar data, created ni methods como en la forma tradicional de Vue. Todo el código que define el estado y las funciones se puede escribir directamente dentro del bloque <script setup>.

  4. Acceso a las variables reactivas:

    • Usamos .value para acceder a los valores de las variables reactivas en city y weatherData, ya que ref envuelve el valor en un objeto.

Si buscas una API gratuita para integrar datos climáticos en tu portafolio web, te recomiendo las siguientes opciones:

1. OpenWeatherMap (Plan gratuito)

  • Descripción: OpenWeatherMap tiene un plan gratuito bastante completo. Puedes acceder a datos actuales, pronósticos de hasta 16 días, y también datos de clima histórico con ciertas limitaciones en cuanto a cantidad de peticiones.
  • Límites: 1,000 llamadas por día.
  • Funcionalidades:
    • Pronóstico actual, por hora (para 4 días) y por día (hasta 16 días).
    • Sin coste adicional para pronósticos y clima actual.
    • Puedes usar la API sin límite de tiempo para pruebas.
  • Enlace: OpenWeatherMap

2. WeatherAPI (Plan gratuito)

  • Descripción: WeatherAPI es otra opción gratuita que ofrece datos de clima actuales, pronósticos, y datos históricos. Es conocida por su facilidad de uso.
  • Límites: 1,000 llamadas diarias de forma gratuita.
  • Funcionalidades:
    • Pronóstico actual, por hora y 7 días.
    • Soporte para múltiples formatos (JSON, XML).
    • Ofrece un plan gratuito que cubre una buena cantidad de funcionalidades para proyectos pequeños.
  • Enlace: WeatherAPI

3. Weatherbit (Plan gratuito)

  • Descripción: Weatherbit es otro proveedor de APIs climáticas que tiene un plan gratuito. Ofrece datos de pronóstico por hora y diario.
  • Límites: 1,000 llamadas por día en su plan gratuito.
  • Funcionalidades:
    • Pronóstico actual, 5 días, y 16 días.
    • Pronóstico por hora (48 horas).
    • También ofrece un historial de datos, pero con limitaciones.
  • Enlace: Weatherbit

4. AccuWeather (Plan gratuito limitado)

  • Descripción: AccuWeather es un proveedor bastante conocido de datos meteorológicos. Ofrece un plan gratuito limitado, pero con funcionalidades básicas útiles para mostrar el clima.
  • Límites: 50 llamadas por día.
  • Funcionalidades:
    • Pronóstico actual y por hora.
    • 15 días de pronóstico a nivel mundial.
  • Enlace: AccuWeather

5. WeatherStack (Plan gratuito)

  • Descripción: WeatherStack ofrece una API simple para obtener datos meteorológicos actuales y pronósticos.
  • Límites: 1,000 llamadas diarias en el plan gratuito.
  • Funcionalidades:
    • Datos meteorológicos actuales.
    • Pronósticos de 7 días.
    • Muy fácil de usar con documentación clara.
  • Enlace: WeatherStack

Conclusión:

De todas estas opciones, OpenWeatherMap y WeatherAPI son dos de las mejores opciones si estás buscando una API gratuita con un buen límite de llamadas. Ambas permiten acceder a datos completos (como pronósticos actuales y por hora) sin requerir un plan de pago, lo cual es ideal para un proyecto de portafolio.

Si tu aplicación va a ser pequeña y no requerirás muchas peticiones al día, estas APIs gratuitas deberían ser suficientes. Solo asegúrate de obtener tu clave de API al registrarte en cualquiera de estas plataformas.

Si necesitas ayuda con la implementación o cómo hacer las peticiones, avísame y estaré encantado de ayudarte.

Para obtener los iconos del clima utilizando la API de OpenWeatherMap, solo necesitas acceder al campo weather de la respuesta de la API. Dentro de weather, hay un campo llamado icon que contiene el código del icono, que corresponde a un archivo de imagen en el servidor de OpenWeatherMap.

Cómo obtener y mostrar los iconos:

  1. La URL del icono es algo así:

    css
    http://openweathermap.org/img/wn/{icon}.png

    Donde {icon} es el valor que obtienes de la propiedad icon dentro del campo weather en la respuesta.

  2. Actualizando tu código para mostrar el icono:

Ejemplo de código:

javascript
const apiKey = "TU_CLAVE_API"; // Reemplaza con tu clave API
const city = "Madrid,es"; // Ciudad de Madrid, España
const url = `https://api.openweathermap.org/data/2.5/weather?q=${city}&appid=${apiKey}&units=metric`; // units=metric para obtener la temperatura en grados Celsius

axios.get(url)
  .then(response => {
    console.log(response.data); // Muestra la respuesta del clima
    
    const iconCode = response.data.weather[0].icon; // Obtiene el código del icono
    const iconUrl = `http://openweathermap.org/img/wn/${iconCode}.png`; // URL para mostrar el icono
    
    // Aquí es donde puedes insertar el icono en el HTML de tu aplicación
    console.log(iconUrl); // Imprime la URL del icono para verificar
  })
  .catch(error => {
    console.log("Error al obtener el clima:", error);
  });

En tu template de Vue.js (si estás usando Vue):

html
<template>
  <div id="app">
    <div v-if="weatherData">
      <h1>Clima en {{ weatherData.name }}</h1>
      <p>Temperatura: {{ weatherData.main.temp }} °C</p>
      <p>Descripción: {{ weatherData.weather[0].description }}</p>
      <p>Humedad: {{ weatherData.main.humidity }} %</p>
      <p>Viento: {{ weatherData.wind.speed }} m/s</p>
      
      <!-- Mostrar el icono -->
      <img :src="iconUrl" alt="Weather Icon" />
    </div>

    <div v-else>
      <p>Cargando...</p>
    </div>
  </div>
</template>

<script>
import axios from "axios";

export default {
  name: "App",
  data() {
    return {
      weatherData: null,
      city: "Madrid",
      apiKey: "TU_CLAVE_API", // Tu clave API aquí
      iconUrl: "", // Aquí se almacenará la URL del icono
    };
  },
  created() {
    this.getWeather();
  },
  methods: {
    async getWeather() {
      try {
        const response = await axios.get(
          `https://api.openweathermap.org/data/2.5/weather?q=${this.city}&appid=${this.apiKey}&units=metric`
        );
        this.weatherData = response.data;

        // Obtener el código del icono y construir la URL
        const iconCode = this.weatherData.weather[0].icon;
        this.iconUrl = `http://openweathermap.org/img/wn/${iconCode}.png`;
      } catch (error) {
        console.error("Error al obtener el clima:", error);
      }
    },
  },
};
</script>

Explicación:

  • iconCode: El valor de icon es un código como 01d, 02n, 10d, etc., que representa el tipo de clima (despejado, nublado, lluvia, etc.).
  • iconUrl: Usamos el código del icono para construir la URL completa y luego podemos insertarlo en el HTML utilizando <img>.

Ejemplo de iconos:

Algunos ejemplos de códigos de iconos que podrías encontrar en la respuesta de la API:

  • 01d: Sol (día)
  • 01n: Sol (noche)
  • 02d: Nubes dispersas (día)
  • 10d: Lluvia (día)

La URL completa para el icono sería algo así:

  • http://openweathermap.org/img/wn/01d.png
  • http://openweathermap.org/img/wn/10d.png

Resultado:

Esto mostrará el icono correspondiente al clima en Madrid (o la ciudad que elijas) y lo actualizará dinámicamente según la respuesta de la API.
